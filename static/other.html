HTTP/1.1 Servers — A Very Basic One-Page Overview

A tiny primer on what an HTTP/1.1 server does, the shape of requests and responses, and a couple of simple examples you can try.

What is HTTP/1.1?

HTTP/1.1 is the application-level protocol used by web servers and browsers for request/response messaging. It standardized persistent connections, virtual hosting via the Host header, and chunked transfer encoding.

  • Persistent connections: keep the TCP connection open for multiple requests unless Connection: close.
  • Host header is required (supports many sites on one IP).
  • Message format: start-line, headers, blank line, optional body.
  • Body delimiting: via Content-Length or Transfer-Encoding: chunked.
  • Status codes: 1xx (info), 2xx (success), 3xx (redirect), 4xx (client error), 5xx (server error).
  • Caching: Cache-Control, ETag, If-None-Match, Last-Modified.
  • Pipelining exists but is rarely used due to head-of-line blocking.

Example request

GET /index.html HTTP/1.1
Host: example.com
User-Agent: curl/8.0
Accept: */*

Notes: The request line is method, path, version. Headers follow, then a blank line. GET here has no body.

Example response

HTTP/1.1 200 OK
Date: Tue, 26 Aug 2025 08:39:00 GMT
Server: ExampleServer/1.0
Content-Type: text/plain; charset=utf-8
Content-Length: 12

Hello World!

After headers and a blank line comes the body. Here the body is 12 bytes: “Hello World!”.

Minimal “hello” with netcat (demo)

This serves a fixed response on port 8080. Open a terminal and run: 

while true; do
  printf 'HTTP/1.1 200 OK\r\nContent-Length: 12\r\nContent-Type: text/plain\r\n\r\nHello World!'
  nc -l -p 8080 -q 1
done

Then in another terminal: curl -v http://localhost:8080/
Tip: Some nc variants use different flags (e.g. -k, -l 8080), adjust as needed.

Chunked transfer (no length known up-front)

HTTP/1.1 200 OK
Transfer-Encoding: chunked
Content-Type: text/plain

B
Hello World!
0

“B” is hex for 11 bytes. Chunks are size line, then data, ending with a zero-size chunk.

Common headers

  • Host — required in HTTP/1.1 requests.
  • Connection — e.g. keep-alive or close.
  • Content-Type — media type of the body.
  • Content-Length — byte length of the body.
  • Cache-Control, ETag, Last-Modified — caching.
  • Range / Accept-Ranges — partial content.
  • Location — with redirects (3xx).

Try a tiny Node.js HTTP/1.1 server

Node’s built-in http module speaks HTTP/1.1 by default.

// save as server.js, then: node server.js
const http = require('http');

const server = http.createServer((req, res) => {
  // Minimal response
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello from HTTP/1.1');
});

server.listen(8080, () => {
  console.log('Listening on http://localhost:8080');
});

Test with: curl -v http://localhost:8080/
